## Summary

The recipe-pipeline edge function handles text, URL, and video sources through adapter → extract → transform → load stages. After resolving the initial extraction failures (field name normalization, quantity string parsing), three areas still need improvement: image handling has no validation or compression, the pipeline only supports one recipe per request, and the quantity parser only recognizes ~20 unit types out of 80+ real-world measurements.

---

## Current State

The extract stage (`supabase/functions/recipe-pipeline/stages/extract.ts`) now includes:
- `findRecipeObject()` — deep-searches parsed JSON for recipe data regardless of nesting
- `normalizeFieldNames()` — maps `recipe_name` → `title`, `steps` → `instructions`, etc.
- `parseQuantityString()` — splits `"8 cloves, minced"` into amount/unit/notes
- `parseIngredientString()` — handles plain string ingredients

Models used: `qwen/qwen-2.5-7b-instruct` (text), `qwen/qwen-2.5-vl-7b-instruct` (vision)

## Scope of Work

### Area 1: Image Handling

#### What Works Today
- Chat UI supports up to 4 images via file picker or clipboard paste (base64 data URLs)
- Text adapter passes images through to extract stage
- Extract stage routes to vision model when images are present
- Video adapter runs vision OCR on video frames

#### Gaps

| Gap | Description | Impact |
|-----|-------------|--------|
| No image validation | No check for corrupt, too-large, or non-food images | Wasted LLM calls, confusing errors |
| No image compression | Full-res photos sent as base64 → huge payloads | Slow uploads, potential timeout, Supabase payload limits |
| Recipe image not saved | Uploaded image isn't stored as the recipe's `image_url` | Recipe card shows no image |
| URL adapter ignores images | `isUrlSource` flag skips vision model even if URL page has recipe images | Missed info from image-heavy recipe sites |

#### Phase 1: Image Validation & Compression (Frontend)
**Files**: `src/components/chat/ChatInterface.tsx`

1. Validate on select: Check file size (max 5MB raw), dimensions, and type
2. Compress before encoding: Use canvas to resize to max 1200px wide, JPEG quality 0.8
3. Show compression indicator on preview

#### Phase 2: Save Uploaded Image as Recipe Image
**Files**: `src/components/chat/ChatInterface.tsx`, `src/services/api.ts`, `supabase/functions/chat-api/index.ts`

1. Upload the first image to Supabase Storage (`recipe-images` bucket) on save
2. Set the public URL as the recipe's `image_url`
3. Pass through so `StructuredRecipeDisplay` shows the image

#### Phase 3: Smarter Vision Prompt
**Files**: `supabase/functions/_shared/recipe-prompts.ts`

Add a dedicated `IMAGE_EXTRACTION_PROMPT` optimized for screenshots, cookbook pages, handwritten cards, and food photos.

### Area 2: Multiple Recipes in One Paste

#### Current Behavior
The pipeline expects one recipe per request. Multiple pasted recipes may get merged, partially extracted, or only the first one returned.

#### Phase 1: Detection (Extract Stage)
**Files**: `supabase/functions/recipe-pipeline/stages/extract.ts`, `supabase/functions/_shared/recipe-prompts.ts`

1. Update `RECIPE_EXTRACTION_PROMPT` to instruct: "If the input contains multiple recipes, return them as an array under a `recipes` key"
2. Update `parseRecipeResponse` to detect array responses
3. Change extract stage return type to `ExtractedRecipe | ExtractedRecipe[]`

#### Phase 2: Pipeline Support
**Files**: `supabase/functions/recipe-pipeline/pipeline.ts`, `supabase/functions/_shared/recipe-schema.ts`

1. Add `recipe_ids?: string[]` and `recipes?: ValidatedRecipe[]` to `PipelineResult`
2. Update `runPipeline` to loop transform + load for each extracted recipe
3. Per-recipe error tracking

#### Phase 3: Frontend Multi-Recipe Display
**Files**: `src/components/chat/ChatInterface.tsx`, `src/components/chat/StructuredRecipeDisplay.tsx`

1. Render a `StructuredRecipeDisplay` card for each recipe
2. Each card has its own independent save button
3. Add a "Save All" button above the cards

#### Edge Cases
- Recipes may be separated by `---`, blank lines, or numbered headings
- Partial extraction: show what was found + note what was missed
- Cap at 5 recipes per request to prevent abuse/timeout

### Area 3: Quantity & Measurement Parsing

#### Currently Supported (~20 units)

| Category | Units | Examples |
|----------|-------|----------|
| Volume | cups, tbsp, tsp, ml, L | `2 cups`, `1/2 tsp` |
| Weight | oz, lb, lbs, g, kg | `8 oz`, `1.5 lbs` |
| Count | cloves, cans, pieces, slices, stalks | `8 cloves`, `2 cans` |
| Size | medium, large, small | `1 medium` |
| Fractions | decimal and `/` notation | `1/2`, `0.5` |

#### Not Handled (~60+ units)

| Category | Missing Units | Examples |
|----------|--------------|---------|
| Containers | block, package, pkg, packet, box, bag, jar, bottle, container | `1 block cream cheese` |
| Informal | dollop, splash, dash, pinch, handful, bunch, sprig, knob | `a pinch of salt` |
| Baking | stick, sheet, layer, round | `1 stick butter` |
| Produce | head, bunch, ear, rib, sprig, leaf/leaves | `1 head garlic` |
| Portions | serving, portion, scoop, drop, squeeze | `1 scoop protein powder` |
| Cooking sizes | whole, half, quarter, cube, wedge, strip, filet/fillet | `1 whole chicken` |
| Unicode fractions | ½, ⅓, ¼, ⅔, ¾ | `½ cup sugar` |
| Ranges | `2-3`, `2 to 3` | `2-3 tablespoons` |
| Written numbers | a, an, one, two, three... | `a pinch of salt` |

#### Phase 1: Expand Unit Regex
**File**: `supabase/functions/recipe-pipeline/stages/extract.ts`

Replace single regex with comprehensive `KNOWN_UNITS` list (~80 unit patterns).

#### Phase 2: Unicode Fractions & Written Numbers
**File**: `supabase/functions/recipe-pipeline/stages/extract.ts`

Add `normalizeUnicodeFractions()` and `normalizeWrittenNumbers()` pre-processors.

#### Phase 3: Range Handling
Parse `"2-3 tablespoons"` → `amount: 2.5, unit: "tbsp", notes: "2-3"` (midpoint, store range in notes).

#### Phase 4: "To Taste" / No-Amount Ingredients
Already handled by fallback path, but should be explicit pattern matching for `"to taste"`, `"as needed"`, `"for garnish"`.

## Acceptance Criteria

- [ ] `parseQuantityString("1 block cream cheese")` returns `{ amount: 1, unit: "block", notes: "cream cheese" }`
- [ ] `parseQuantityString("½ cup sugar")` returns `{ amount: 0.5, unit: "cup", notes: "sugar" }`
- [ ] `parseQuantityString("a pinch of salt")` returns `{ amount: 1, unit: "pinch", notes: "of salt" }`
- [ ] Images over 5MB are rejected with user-facing error
- [ ] Images are compressed to < 500KB before sending to pipeline
- [ ] Pasting 2 recipes returns 2 separate recipe cards with independent save buttons
- [ ] Pasting 5+ recipes caps at 5 and shows a note
- [ ] Saved recipe from image upload shows the image on the recipe card

---

## Notes

- The Qwen 2.5 7B model does not reliably follow the prompt's exact field naming (uses `recipe_name` instead of `title`, `quantity` instead of `amount`+`unit`). The normalization layer in `extract.ts` handles this, but future model upgrades may improve compliance.
- `response_format: { type: "json_object" }` is passed to OpenRouter but the model still wraps responses in markdown code blocks. The parser handles this via regex fallback.
